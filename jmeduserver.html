학원의 문제점 및 요구사항 분석

  • 자녀가 학원에 잘 도착했는지 알 수 없다.
  • 수업 도중 무단 조퇴하는 학생이 있다.
  • 출석부 관리를 자동화하고 싶다.
  • 수업별로 학생, 학부모에게 단체 공지를 하고 싶다.
  • 강의실 수 부족으로 일정 협의가 어렵다.
  • 학생 개인정보 접근 로그를 남겨 법적 대응이 가능하도록 한다.

제공 솔루션

  • 카카오 알림톡 및 SMS로 등/하원 통지
  • 수업 시작 시 키오스크에서 등원 처리
  • 무단 등원 적발 및 알림
  • 출석부 자동 관리 및 확인 기능
  • 수강 학생 대상 공지 전송 기능
  • 개인정보 열람/수정 로그 기록
  • 수업 중복 방지 일정 등록 시스템

시스템 구성 다이어그램

  1. 학부모에게 SMS 또는 알림톡 전송
  2. 학생이 QR코드 인식
  3. 등/하원 결과 안내
  4. 알리고 컨테이너로 데이터 전송
  5. 처리 결과 키오스크에 전송
  6. 관리 페이지 제공
  7. React 페이지로 요청
  8. 학부모 연락처로 메시지 전송
  9. 직접 입력한 메시지 전송
  10. 학생 이름 반환
  11. CRUD 작업 수행

📦 Python Module GitHub: CutTheWire/ClassLinker

Docker 컨테이너 구성

  • NginX: 모든 요청 분배
  • Aligo (Python): QR 인식 → 알림 전송
  • React: 키오스크 및 관리자 페이지 제공
  • Node.js: 로그인, 수업관리, 공지, 개인정보 로그
  • MySQL: 전체 데이터 저장

데이터베이스 구성

  • school: 학교 정보
  • student: 학생 정보
  • teacher: 강사 정보
  • attendence_log: 등/하원 기록
  • admin_log: 개인정보 접근 기록
  • teacher_attend_log: 강사 출퇴근 기록
  • subject: 과목 정보
  • plan: 시간표
  • student_subject: 수강 정보
  • subject_executed: 수업 실행 기록
  • subject_executed_attenders: 참여 학생 기록
  • permissions: 권한 정보
  • serverconf: 서버 설정

김준건의 포트폴리오

학생관리시스템 서버 및 주요 기능 소개

📄 프로젝트 개요

JMEDUSERVER는 학생 정보 관리와 출결, 공지, 개인정보 로그 등 학원 운영에 필요한 다양한 기능을 제공하는 서버 시스템입니다.

GitHub 저장소 바로가기

🎯 이 프로젝트의 목적

  • 계약대상인 제이엠에듀국영수논술전문학원에 학생 출결관리를 위한 솔루션을 제공한다.
  • 등/하원 등록 시 학부모에게 즉시 문자메세지를 보내 학부모를 안심하고 학원에 보낼 수 있게 한다.

🏆 프로젝트를 통해 내가 이룬 목표

  • 외부 API 사용 가이드를 참고하여 사용하기 (알리고 문자 API)
  • 개발자 입장에서는 생각하기 어려운 실제 고객의 요구사항과 피드백을 수용하여 개발하기
  • Node.JS 개발 환경 훈련하기
  • 명세서를 통해 프론트엔드 개발자에게 보기 쉬운 API 가이드라인 제공하기

주요 화면

🎬 시연 영상

고객의 요구사항 분석

  • 자녀가 학원에 잘 도착했는지 알 수 없다.
  • 수업 도중 무단 조퇴하는 학생이 있다.
  • 출석부 관리를 자동화하고 싶다.
  • 수업별로 학생, 학부모에게 단체 공지를 하고 싶다.
  • 강의실 수 부족으로 일정 협의가 어렵다.
  • 학생 개인정보 접근 로그를 남겨 법적 대응이 가능하도록 한다.

제공 솔루션

  • 카카오 알림톡 및 SMS로 등/하원 통지
  • 수업 시작 시 키오스크에서 등원 처리
  • 무단 등원 적발 및 알림
  • 출석부 자동 관리 및 확인 기능
  • 수강 학생 대상 공지 전송 기능
  • 개인정보 열람/수정 로그 기록
  • 수업 중복 방지 일정 등록 시스템

시스템 구성 다이어그램

  1. 학부모에게 SMS 또는 알림톡 전송
  2. 학생이 QR코드 인식
  3. 등/하원 결과 안내
  4. 알리고 컨테이너로 데이터 전송
  5. 처리 결과 키오스크에 전송
  6. 관리 페이지 제공
  7. React 페이지로 요청
  8. 학부모 연락처로 메시지 전송
  9. 직접 입력한 메시지 전송
  10. 학생 이름 반환
  11. CRUD 작업 수행

데이터베이스 구성

  • school: 학교 정보
  • student: 학생 정보
  • teacher: 강사 정보
  • attendence_log: 등/하원 기록
  • admin_log: 개인정보 접근 기록
  • teacher_attend_log: 강사 출퇴근 기록
  • subject: 과목 정보
  • plan: 시간표
  • student_subject: 수강 정보
  • subject_executed: 수업 실행 기록
  • subject_executed_attenders: 참여 학생 기록
  • permissions: 권한 정보
  • serverconf: 서버 설정
